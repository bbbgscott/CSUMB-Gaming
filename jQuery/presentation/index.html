jQuery 101: selectors

$('#myid');    //ID (must be unique)
$('.myClass'); //class
$('input[type=text]'); //attributes (can be slow)
$('.myClass p'); //any CSS compound selector

jQuery 101: special selectors

$('a:not(.myClass)'); //Any link without the class of "myClass"
$('input:checked'); //Any checkbox which is checked
$('div > p'); //Any "p" that's the child of "div"
$('p:contains("test")'); //Any p that contains the word "test"
$('tr:even'); //Any "tr" that is even compared to siblings
$('tr:odd'); //Any "tr" that is odd compared to siblings

Attributes:

Attributes are easily manipulated with the .attr() function:

$('a.myClass').attr('href'); //get href
$('a.myClass').attr('href', 'index.html'); //set href
//Set a list via classes
$('a.myClass').attr({href : 'index.html',
					 rel  : 'external'});
//Set via functions
$('a.myClass').attr({href : 'index.html',  
					 rel  : function(idx, rel) {
					 	//OMG!
					 	return 'external';
					 }
				   });

"Bind" vs. "Live"

Take this code:

$('.clickme').bind('click', function() {
  // Bound handler called.
});

But if you do this:

$('body').append('
Another target
');

Then the event won't fire. However, "live" will watch for that:

$('.clickme').live('click', function() {
  // Bound handler called.
});

"AJAX" is not really "AJAX"

It can actually retrieve anything, mostly HTML or JSON

  • Same-origin policy (why we can't do this locally)
  • Anything can be returned, really.
  • There's a single ".ajax()" method, but some other helpful ones:
    • .get()
    • .post()
    • .getJSON()
    • .getScript() - helpful for auto-loading

Standard GET Request

Note that we are using this as a method on the jQuery object itself.

$.get('something.php', function(data) {
	$('body').append(data);
});
